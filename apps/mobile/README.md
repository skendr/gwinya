# Gwinya — mobile (Expo / iOS)

Native client for Gwinya, sharing the Next.js backend (Supabase + the `/api/*`
routes) and the framework-agnostic TypeScript in `packages/shared` (lesson
content, domain types, red-flag detector, date + streak helpers).

## Setup

```bash
# from the repo root
pnpm install
cp apps/mobile/.env.example apps/mobile/.env   # fill in the values
```

`.env` (all values are public — never put AI/service-role keys here):

- `EXPO_PUBLIC_SUPABASE_URL` / `EXPO_PUBLIC_SUPABASE_ANON_KEY` — same Supabase
  project as the web app, anon key.
- `EXPO_PUBLIC_API_BASE_URL` — deployed Next.js URL (e.g. `https://gwinya.vercel.app`),
  used by the lesson chat.

## Run

```bash
pnpm --filter @gwinya/mobile start        # Metro; press i for the iOS simulator
pnpm --filter @gwinya/mobile run:ios      # native build via xcodebuild + simulator
pnpm --filter @gwinya/mobile typecheck    # tsc --noEmit
```

The native `ios/`/`android/` projects are generated by `expo prebuild`
(Continuous Native Generation) and are gitignored.

## Notes

- Design tokens are ported from the web `globals.css` into `src/theme.ts`.
- `babel.config.js` rewrites supabase-js's inert OpenTelemetry dynamic import
  so Hermes can compile the bundle.
- Auth uses the Supabase JS client with a chunking SecureStore adapter and PKCE
  (magic-link deep links land on the `auth-callback` route).
